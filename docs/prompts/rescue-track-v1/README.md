# 补救轨迹第一版实现提示词

把下面提示词粘贴到新的 Codex 对话中执行。第一版只做手机端、高德地图、补救 GPX 主流程；电脑浏览器访问手机端口属于后续版本。

```text
你在 D:\code\gpx-recorder 这个 Android/Kotlin 项目里工作。请先阅读仓库 README、AGENTS.md/用户规则、现有 GPX 数据模型、路线列表、录制配置页、详情页、高德地图控制器和导出逻辑，再实现“补救轨迹”第一版。

重要约束：
- 默认用简体中文沟通。
- 先不要处理 Google Maps，所有新增补救轨迹能力只要求高德可用；Google 侧可以保持原状或显示不支持。
- 不要做电脑浏览器访问手机端口，那是后续版本。
- 现有正常 GPS 录制、路线列表、详情页、导出 GPX/GeoJSON 功能不能回归。
- 补救轨迹每一步操作都要落库保存，防止中途退出丢失。
- 最后“保存”不是第一次保存，而是关闭草稿并生成正式 GPX 记录。

功能目标：
做一个“补建/补救轨迹”功能，用于用户忘记开 GPS 记录时，根据手动标点、手机照片/视频的经纬度和拍摄时间、高德路线规划，生成一条可导出的 GPX 轨迹。这个 GPX 后续主要用于给相机、全景相机照片和视频做经纬度标记，所以时间轴可靠性很重要。

入口设计：
- 主页右下角 FAB 仍然进入现有的“新建路径/开始录制配置”页面，也就是当前 RecordingConfiguratorModal / config_dialog 这一套入口，不要删除或替换它。
- 现有开始录制页面现在比较空，需要在这个页面内部扩展布局：
  - 上半屏新增“补救轨迹”入口，例如“补建轨迹 / 忘记记录时手动创建 GPX”。
  - 下半屏保留原来的正常 GPS 录制配置功能，包括标题、记录间隔、开始按钮等。
- 用户点击下半屏的原有开始按钮时，行为必须和现在完全一致：创建或继续正常 GPS 录制。
- 用户点击上半屏“补救轨迹”入口时，进入新的补救轨迹草稿流程。
- 不要创建一个额外的中转页面来替代原配置页；是在原配置页上方加入口，把原有功能压到下半部分。

核心流程：
补救轨迹做成分步骤流程，并自动保存草稿状态。

步骤一：画草图
- 用户进入补救轨迹后创建一个草稿。
- 地图使用高德。
- 用户可以搜索国内地点，用于快速定位地图视野。
- 用户点击地图添加锚点。
- 第一个点必须弹窗选择时间；如果取消，则删除这个点，不允许继续。
- 后续点点击地图即可添加，默认不要求填写时间。
- 点可以拖拽调整位置。
- 点击点可以修改时间、删除点。
- 可以导入手机照片/视频。
- 媒体点以媒体 EXIF/元数据里的 WGS84 经纬度和拍摄时间为准，只要导入就当作正确数据。
- 高德展示和编辑时才做 WGS84 <-> GCJ-02/高德坐标转换；入库和最终 GPX 使用 WGS84。
- 媒体时间落在已有起终点时间范围内，则按时间插入中间。
- 媒体时间早于当前起点或晚于当前终点时，导入时提示用户：该媒体时间超出当前路线时间范围，是否继续；继续后插到前面或后面。
- 如果媒体时间落在 A-B 时间段，但媒体坐标离原本 A-B 草图很远，也直接插成 A -> 媒体点 -> B，不要吸附到原线。
- 手动无时间点允许存在，后续用相邻有时间点插值。
- 完成步骤一时必须确保至少有两个点，并且起点和终点都有时间；如果最后一个点没时间，点击下一步时要求填写终点时间。

步骤二：路径规划
- 草图确认后，按相邻锚点拆成多段。
- 每段选择交通方式：驾车、步行、骑行、直线。
- 第一段选择后，后续段可以沿用上一段方式，也可以重新选择。
- 对非直线段调用高德路线规划，展示高德返回的候选路线，让用户选择真实走过的路线。
- 每段的交通方式、候选路线选择结果都要保存到草稿。
- 不要在步骤一画点阶段就生成中间点；只在步骤二/三之后生成。

步骤三：生成路径点
- 用户确认后，根据每段已选择的规划路线生成 GPX 轨迹点。
- 生成点按时间间隔撒点，默认 5 秒。
- 提供选项：1 秒、5 秒、10 秒、30 秒。
- 每段根据两端时间算生成点数量，再沿选中的路线按距离比例取坐标。
- 媒体点和手动带时间点是强锚点，必须保留，不被自动生成点覆盖。
- 如果一段时间过短但距离过长，提示速度异常，但第一版不强制阻止生成。
- 如果高德规划失败，允许用户改为直线，或返回重新选择。

步骤四：预览并保存
- 展示最终生成的轨迹预览。
- 用户确认保存后，生成正式 GpxContent/Track/Segment/TrackPoint，进入现有路线列表。
- 生成后的正式路线要能复用现有详情页、导出 GPX/GeoJSON、分享/保存能力。
- 草稿状态标记为 COMPLETED 或归档，不再作为未完成草稿提示。
- 如果用户中途退出，再进入补救轨迹时应能恢复未完成草稿，或提示“继续上次补建轨迹 / 新建”。

草稿模型建议：
新增独立草稿数据，不要直接把半成品写成正式 GpxContent。
至少需要保存：
- draftId
- title
- currentStep
- createdAt/updatedAt
- anchors：坐标 WGS84、时间可空、来源 manual/media、媒体 URI/名称可选、顺序、是否锁定
- segments：fromAnchor、toAnchor、travelMode、selectedRoute/polyline、规划状态
- generationConfig：点间隔秒数
- generatedPreviewPoints 可选

状态建议：
- DRAWING：画草图
- PLANNING：逐段规划路线
- GENERATED：已生成预览
- COMPLETED：已生成正式 GPX
- ABANDONED：放弃

时间规则：
- 起点必须有时间。
- 终点完成草图时必须有时间。
- 媒体点必须使用媒体时间。
- 所有有时间的锚点在路线顺序上必须递增；如果倒序或相等，生成前提示用户修改。
- UI 显示本地时间，GPX 导出保持现有 DateTimeFormatHelper/GPX 时间格式兼容；注意后期 geotag 软件兼容性。

媒体导入：
- 第一版优先支持手机本地照片；视频能读则读，读不到就提示不可用。
- 有经纬度和拍摄时间才能作为媒体锚点导入。
- 没有经纬度或没有拍摄时间的媒体，列为不可导入并提示数量。
- 媒体坐标按 WGS84 保存。
- 不要擅自纠错媒体 GPS 或时间。

验收标准：
- 正常录制入口仍能使用。
- 可以从主页 FAB 进入现有新建路径页面，并在该页面上半屏选择补救轨迹。
- 该页面下半屏原有 GPS 录制配置和开始行为保持不变。
- 补救轨迹可以创建草稿、添加起点并强制选择时间、继续添加点、拖拽点、编辑点时间、删除点。
- 可以导入带 GPS 和拍摄时间的手机照片作为媒体点，并按时间插入路线。
- 可以按段选择交通方式和高德候选路线。
- 可以按 5 秒默认间隔生成预览轨迹点。
- 可以保存为正式 GPX 记录，并在现有列表中打开、导出。
- 中途退出后草稿不丢失。
- 运行必要的 Gradle 构建/测试；如果是 Android 项目，完成后按项目规则编译，并检查 MuMu 模拟器端口，若开启则安装到 MuMu 模拟器。

实现要求：
- 优先复用现有 GpxContent、Track、Segment、TrackPoint、ExportFragment、AmapCoordinateConverter、AmapRouteMapController 中已有能力。
- 复杂逻辑从 Fragment 中拆出，避免 UI 文件过大。
- 不要大规模重构无关模块。
- 新增字符串尽量中文优先或保持项目现有多语言结构一致。
- 完成前说明改动文件、验证命令和结果。
```
